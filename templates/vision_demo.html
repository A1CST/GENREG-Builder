Two ways to build larger gradient-free vision systems

Both models below are bred by selection — no gradients, no backprop — and grounded in pixels. One knows shapes (10 classes), one knows letters (26). Two composition strategies turn them into one model that knows both: MERGE two independently trained models (union), or GROW one model into the new task (continued training). They answer different questions — can trained models be merged without retraining? and can a trained model expand into a new task without starting over? Every number is test-touched-once over 36 classes (chance ).

Strategy 1 · Merge

Merge a shapes model + a letters model

Run each frozen bank in its own basis at its native scale, concatenate the two feature vectors, and fit one closed-form ridge classifier over 36 classes. Each bank stays frozen — the only learned component is that linear readout over the concatenated activations — so there is no interference to manage and no genome is retrained.
Strategy 2 · Grow

Grow the shapes model until it reads letters

Start from the frozen shape genomes, then keep evolving new genomes on the 36-class task. The previously evolved genomes stay frozen; each new genome is scored only for the improvement it adds beyond the frozen base — so prior capabilities are preserved while the feature bank extends (no catastrophic forgetting to guard against). One model grows: no separate letter model is ever trained. Because the shape features already transfer to letters, training starts well above chance and climbs.

When to use which

Merge — fuse two models

Needs a shapes model and a letters model, each already trained. Fusion is instant (one head solve) and the two perceptions are complementary, so fused beats either bank. Best when you already own both experts and want them under one readout.

Grow — extend one model

Needs only the shapes model. It keeps evolving new genomes for the new classes, so it can learn letters without anyone ever training a letter model. Best when the second expert doesn't exist yet — the model extends itself.

What this suggests

Taken together, the observations point at a property of the representation itself, not of shapes and letters specifically: independently evolved feature banks can be merged under one readout; a frozen bank can be extended without disturbing what it already computes; the evolved features remain useful on a related domain they were never trained on; and adding capacity does not require relearning the old representation.

That is a composability / modularity property — closure under composition — which gradient-trained feature extractors often lack. Whether it holds on harder datasets than shapes and letters is the open question; here it is a clean, measurable existence proof rather than a claim.
