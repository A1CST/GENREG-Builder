The temporal radial stack — naming the animation from its motion

The animations below are the dataset. The model watches a 6-frame window of one of them and must say which animation it is — but the shape riding the path is randomized, the window starts anywhere in the clip, and the whole path is shifted by a random offset. Shape is a decoy and position is decorrelated: only the motion between frames names the animation (10 classes, chance 0.10). A twin model trains on the same sequences with the labels flipped — name the shape, with the motion as the decoy — so the two stacks show where evolution puts the work when the answer lives in a frame versus between frames. The current best radial setup (universal grammar, emergent-cap stacked spaces, no gradients) is wired to both below; results update live from the run exports.

The wiring
frames

Every frame is a row

The environment holds all 6 frames of every sequence as ordinary images (patch-PCA maps, label-free). Nothing temporal is designed in — the frames are just more data at this stage.

R0

Per-frame perception

Grammar-v2 spatial genomes evolve as usual; a genome's fitness column is its per-frame scalar averaged over the sequence. R0 becomes shape / position detectors — it cannot see motion.

hand-off

Time becomes channels

Each frozen R0 genome emits its spatial grid map for every frame; the R1 bank is laid out (genome × frame). The universal grammar's folds, shifts and moment stats now compose across time for free: the centroid's trajectory across frames is the path.

R1+

Temporal composition

Standard emergent-cap stacked spaces over the temporal bank — depth, size and primitive usage decided by the energy economy, not by hand. Closed-form head; test touched once.

What a temporal genome can say
# expressible in the universal grammar over the (genome × frame) bank:
falling  = comy-pool( shape-detector @ frame5 ) − comy-pool( shape-detector @ frame0 )
turning  = |com( detector @ f4 ) − com( shift(detector @ f2, dy,dx) )|
# zigzag reverses comy, loop bends comx AND comy, line holds comy flat —
# the same primitives that composed space, composing time

Scaling & robustness — pushing the model

The base task is solved; now we make it harder one distractor at a time and watch where it breaks. Each experiment is a module appended below — nothing above is replaced. Module 3 · Continue training — repairing the weakness without a restart
Module 2 exposed a real limit: the color model fails on inverted B&W. So instead of retraining from scratch, we continue training on top of the exact same model — its frozen genome banks are the warm base, and we keep stacking new spaces on a color+inverted mix. The evolved 40K-param model is kept; only new spaces and the closed-form head adapt. This is the first test of the warm-start method itself (anim_continue.py).

Module 5 · Resolution scaling — cranking the pixels
Everything so far ran at 32×32, where small shapes are a 3-pixel smudge (Module 4). Since the stack above R0 is resolution-invariant (the grid hand-off pools every space to a fixed size), we can just retrain at higher resolution and give the small shapes real pixels. Same scenes, sampled at 32 / 48 / 64. Does the small-shape floor recover (it was resolution-bound) or hold (it was model-bound)? anim_res.py.

Module 6 · One model, many resolutions
Module 5 trained a separate model per resolution. Can one model see all of them? The stack above R0 is resolution-invariant (the grid hand-off pools to a fixed size), so it should be possible. First we check whether a single-resolution model already generalizes; then we continue-train it on a resolution mix (merging each resolution's frames in the grid space) and test across resolutions — from a high base down, and from a low base up. anim_multires.py, anim_continue_res.py.

Attention & control — a little agent

The clutter wall showed the stack can't attend — pick one object out of several. So we build attention from the ground up, on a cursor. A red cursor dot is pinned to a shape; the plan is three models: 1) tracking — follow the cursor and output its location; 2) control — hand the cursor to the model and train it to move over the correct shape (the first autonomous action); 3) action — choose the shape (circle or square). Same gradient-free regression training; the tracker below is Model 1.

Model 1 · Tracking — the model watches the red cursor glide (pinned to a moving shape) amid colored distractor shapes and outputs its position each frame. The target shape is a random color, drawn from the same palette as the distractors — so nothing but the red cursor on it marks it as the thing to follow (an earlier version drew the target white, a shortcut that let the model skip the cursor). The green crosshair is the model's prediction; it should stay on the red dot and ignore the clutter.
