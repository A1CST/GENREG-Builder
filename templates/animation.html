The temporal radial stack — growth as a class

Six-frame animations of a shape that grows at one of four rates and either drifts or holds still. Start size, position, and shape are all random, so no single frame can answer — the label (rate × motion, 8 classes) exists only in what changes between frames. The current best radial setup (universal grammar, emergent-cap stacked spaces, no gradients) is wired to it below; results update live from the run export.

The wiring
frames

Every frame is a row

The environment holds all 6 frames of every sequence as ordinary images (patch-PCA maps, label-free). Nothing temporal is designed in — the frames are just more data at this stage.

R0

Per-frame perception

Grammar-v2 spatial genomes evolve as usual; a genome's fitness column is its per-frame scalar averaged over the sequence. R0 becomes size / shape / position detectors — it cannot see rate.

hand-off

Time becomes channels

Each frozen R0 genome emits its spatial grid map for every frame; the R1 bank is laid out (genome × frame). The universal grammar's folds, shifts and moment stats now compose across time for free: com(g@f5) − com(g@f0) is growth.

R1+

Temporal composition

Standard emergent-cap stacked spaces over the temporal bank — depth, size and primitive usage decided by the energy economy, not by hand. Closed-form head; test touched once.

What a temporal genome can say
# expressible in the universal grammar over the (genome × frame) bank:
growth  = |comx-pool( blob-detector @ frame5 ) − comx-pool( blob-detector @ frame0 )|
drift   = |com( detector @ f4 ) − com( shift(detector @ f1, dy,dx) )| # same primitives that composed space, composing time
The task, live
Four sample sequences looping (drawn from the same generator rules as the dataset): ring/blob, random start size and position, one of four growth rates, drifting or static. The label under each is what the model must recover from the frames alone.
