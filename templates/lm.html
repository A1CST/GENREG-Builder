The radial stack on language — sequential composition, isolated

One variable at a time. The model gets character identity for free — a 6-char context enters as (position × char) one-hot channels — and must predict the next character (27 classes). The anchor is a closed-form ridge on those raw channels with no genomes at all: that additive model literally contains the bigram table but cannot represent interactions ("t then h"). Genome spaces only earn fitness for what they add beyond it — conjunctions composed across positions. If the stack climbs past the one-hot ridge and the bigram ceiling toward the trigram mark, sequential composition was learned, not assumed. (Stage 2 — glyph frames, vision + language at once — already ran: 0.2950 vs bigram 0.2894; preserved in lm_radial_glyph.json.)

The wiring
input

Character IDs, one-hot

A window is (position × char) = 162 binary channels. No pixels, no perception problem — identity is given so only composition is under test. Test windows come from a disjoint corpus region.

anchor

The no-genome ridge

A closed-form head on the raw channels. Additive by construction: it holds the full bigram table plus per-slot unigram tilts, but no interactions. Every genome's fitness is measured as residual gain over this anchor.

genomes

Conjunctions across positions

Vec-grammar genomes fold channels with ×, min, |−| and gates: min(t@5, h@6) is a trigram detector. Every space also keeps the raw one-hot skip bank, so identity stays reachable at any depth.

stack

Emergent-cap spaces

Frozen conjunctions become channels for the next space — compositions of compositions. Depth decided by the cap pressure, not by hand. Closed-form 27-way head; test touched once.

What a compositional genome can say
# expressible in the vec grammar over [raw one-hots | prior spaces]:
trigram   = min( t@slot5 , h@slot6 ) # "..th" → e
4-gram    = min( genome_trigram(x) , space@slot4 ) # " th" — a composition OF a composition
# the additive anchor cannot write either of these; the genomes can
