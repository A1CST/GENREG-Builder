Encoder trainer

Trains a standalone context encoder on the corpus and saves it to the encoder store (runs/encoder/…, also visible on the runs dashboard). It uses the sidebar's Compute, Embed dim, Context dim, Context window, Training samples, Population, Mutation and Elite settings. Once saved, pick it under Encoder in the sidebar so tree runs reuse it instead of re-evolving one every time.

the encoder is the binding constraint — 200 gens + pop 300 + 16k samples was worth ~+1 pt over 60/150/8k (at ~4× encoder time)

ridge = held-out accuracy of a closed-form linear readout — aligned with ridge-seeded leaves, can't win by overfitting centroids. Measured: −0.08 → +2.65 pts vs bigram
re-shuffle the ridge fit/val split every generation so selection can't overfit a fixed validation half. Measured: +4.38 → +4.58 vs bigram
grow/shrink the output dimension count (4–4× the sidebar Context dim); pair with time/diversity for shrink pressure. Stays on CPU
fitness ÷ (1 + active-weights/budget) for ALL gens (uses the speed time budget below)

start the byte-embedding table as a PCA of each byte's context profile (bytes used in similar contexts start near each other) instead of random — the embedding table is otherwise a neutral plateau evolution never organizes (check the 3D plot on /runs)
genomes more DIFFERENT from the rest of the population get a fitness boost at selection (fitness × (1 + strength × novelty)) — fights premature convergence; champion is still ranked on raw fitness
