reverse-diffusion steps; one independent population per level, cleanest→noisiest
Diffuse (the diffusion reverse process): each champion learns ONE small step (noise level k→k−1), so no single genome can finish and the reverse walk starts from noise and moves visibly closer to the image each step — every genome doing an easy job. Denoise: each champion jumps straight to clean from its level; use single-shot (below) — sharper final, but no step-by-step process.
(diffuse mode) train each champion on the ACTUAL output of the champions above it — the real image the walk hands it at inference — instead of a freshly-noised image. Removes the train/inference mismatch so the walk stays monotonic to the end instead of drifting back up. Measured: final L1 0.23→0.12, improvement +25%→+49%. Slightly slower (each step runs the partial walk).
only used in Denoise mode. Single-shot applies just the denoiser matched to the input's noise level — optimal there: only that step sees the true observation. DDIM carries the residual (rescaled) so multi-step at least doesn't collapse; Ancestral injects fresh noise each step → a chain of local averagers regresses to a grey blob.
the "really easy task": each genome sees a (2h+1)² local patch and predicts the clean centre pixel. 1 → 3×3
tiny MLP width — genome is patch → tanh(H) → 1
fitness = denoising MSE over a FRESH minibatch each generation — this stochastic averaging is what makes the population generalize instead of memorizing one image
stop a level once its best MSE hasn't improved for this many generations
each genome carries its own evolving step size (ES style)
