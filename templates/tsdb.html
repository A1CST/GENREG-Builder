TSDB.md v1.3

TSDB is a small Float64 block store for numeric row data. It appends binary rows into physical <id>.fabf files and keeps metadata in a <name>.json manifest. Not a DBMS, ORM, query language, or market-data engine — it stays dumb, compact, append-oriented, and Float64-based.

The vocabulary: a file is a logical group, a key is a schema lane inside it (km = [fieldMap, rowLength]), and a block is the metadata for one bulk append — { info, begin, ends, count, offset }. The offset/count let a reader pull only the bytes it needs instead of the whole file.

This page runs an in-browser port of the store (tsdb.js): the physical layer is an in-memory byte buffer instead of a .fabf on disk, but the manifest, keys, schemas, sorted-block insert, and serialize/parse are the same. It is fed real numbers from the frozen pipeline champions — pick the set below.

Push limits

Hammer the store with a large synthetic load: many blocks appended in random begin-time order to stress the sorted binary-search insert (bound) and the growing physical buffer. Every value is a reproducible hash, so read-back is checked against a recomputed checksum — real integrity, not just a byte count. Runs on the main thread; big loads will pause the tab briefly.
